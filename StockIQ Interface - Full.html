StockIQ Interface Specification

Listed below are the tables used for importing data into StockIQ. Create pipe ("|") delimited files that match the columns and types specified here, then StockIQ can import the files. Name each file the same as the table for which it will be imported, with a ".txt" extension, such as "Item.txt" or "ItemSite.txt".

If you have a null value, or do not have data for a field that can be null, just leave the field empty in the file. For example, if you do not have a value for a field called "FieldB", your file would look like "FieldA||FieldC|FieldD..." and so on.

These files should be generated each day. Typically, they are loaded and processed once per day, in the early morning hours, although more frequent refreshes can be scheduled.

Most commonly, you will want to generate complete files with each refresh, though StockIQ does support partial data imports. Contact StockIQ for more information on how to synchronize this with the StockIQ Application server so that the files are properly processed.

The interface tables are broken up into three main groups, called schemas:

dati - these are tables for data import, usually from your ERP or WMS. It is expected that these will be imported every day, or one some normal, repeating schedule.

impt - these are tables for data that is normally kept in StockIQ, for which you want to do occasional imports. Examples of this include historical itemsite data, forecasts, event imports and more. These may configured to import every day, infrequently, or even one time.

dato - these are tables for data outputted by StockIQ, such as purchase orders, transfer orders, work orders, and forecasts.
